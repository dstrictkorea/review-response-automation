# OPERATIONAL_GUIDE.md — ARTE Review Desk 운영 가이드
> 최종 갱신: 2026-06-11 · 대상: 지점 마케팅 담당자 / 지점장(Director) / 본사 운영자
> 이 시스템은 **자동 게시 봇이 아닙니다.** 모든 답변은 사람이 승인한 뒤 수동으로 게시합니다.

---

## 1. 하루 운영 흐름 (담당자 기준)

```
① 리뷰 인입 확인        /reviews — 신규(new)·승인대기(pending_approval) 필터
② AI 초안 확인          ai_done = 알고리즘이 자동 작성 완료 (안전 리뷰만)
③ 격리 리뷰 우선 처리    pending_approval = 사람 검토 필수 (아래 §3)
④ 초안 수정·승인        상세 페이지에서 편집 → 역할별 액션
⑤ 게시                 답변 복사 → 해당 플랫폼에 직접 붙여넣기 → "게시 완료" 표시
⑥ 아카이브              처리 완료 이력은 /archive 에 영구 보존
```

## 2. 상태(status) 의미

| 상태 | 의미 | 담당자가 할 일 |
|---|---|---|
| `new` | 인입됨, 분류 전/모호 | AI 생성 실행 또는 직접 작성 |
| `ai_done` | 알고리즘이 안전 판정 + 정적 답변 자동 완성 | 내용 확인 후 게시 (수정 가능) |
| `pending_approval` | **격리** — 긴급/불만/모호/질문 포함 | 반드시 검토·수정 후 승인 |
| `published` | 게시 완료 | — |
| (soft deleted) | 보관 삭제 (감사 추적 보존) | /archive에서 조회 |

## 3. 어떤 리뷰가 자동으로 격리되는가

다음은 시스템이 **절대 무승인 자동완료하지 않는** 리뷰입니다:

1. **EMERGENCY** — 부상/사고, 법적 위협(소송·고소·변호사), 환불·보상 요구, 언론 제보 위협,
   CCTV/개인정보 요구, 직원 징계 요구, 차별/혐오, 낙상. → 건조 사과 초안만 제공, 관리자 검토 필수.
2. **★1–2점 + 긍정스러운 문장** — 별점과 본문이 충돌하면(예: "전시는 괜찮았는데…")
   미탐지 불만 가능성으로 보고 격리합니다.
3. **서비스 질문 포함** — "유모차 대여 되나요?", "주차 되나요?" 등은 `[질문]` 태그가 붙고
   격리됩니다. **AI는 시설 정보를 지어내지 않습니다 — 사람이 정확한 정보로 답하세요.**
4. **★4–5점인데 불만 태그 2개 이상** — 비꼬기(사캐즘) 의심.
5. **욕설/독성 표현** — Tier 2(critical 지목·아동부상 등)와 Tier 3(미등록 극단 욕설)은 격리.
   Tier 1 일반 비속어는 비즈니스 언어로 순화되어 처리됩니다.

## 4. AI가 절대 하지 않는 것 (안전 규칙 — 코드로 강제)

- ❌ 환불·보상·무료티켓·할인 약속
- ❌ 법적 책임 인정
- ❌ CCTV 확인 약속
- ❌ 직원 징계·처벌 약속
- ❌ 어떤 형태든 자동 공개 게시

초안에 위 표현이 감지되면 `forbidden_check` 플래그가 켜지고 승인 전 수정이 강제됩니다.
**직접 수정할 때도 위 표현을 넣지 마세요** — 보상 관련 문의는 "담당 팀이 별도 연락"
패턴만 사용합니다 (예문: 상세 페이지의 EMERGENCY 초안 참고).

## 5. 역할별 액션

| 역할 | 가능한 액션 |
|---|---|
| `marketing_staff` | 담당 지점 리뷰 처리 · low 리스크 직접 게시 · 🚨 관장 결재 요청 |
| `director` | 지점장 전결 승인·게시 · 본사(HQ) 이관 · 고위험 격리 큐 처리 |
| `admin` | 전 지점 + 사용자/지점 관리 + 규칙 설정(/settings/rules) |

지점 접근 권한은 `profiles.assigned_branches` 기준으로 앱 레벨에서 강제됩니다(fail-closed).

## 6. 다국어 답변

- 리뷰 언어가 **한국어/영어/일본어/중국어/스페인어/러시아어/아랍어/힌디어/필리핀어**(9개)면
  해당 언어 네이티브 답변이 자동 생성됩니다. 지점명·작품명(WAVE, FOREST 등)은 고유명사로 유지됩니다.
- 그 외 언어(독일어·프랑스어 등)는 한국어 폴백 초안이 생성되므로 **격리 여부와 무관하게
  게시 전 반드시 해당 언어로 직접 작성/번역하세요.**
- 한국어 답변의 조사(을/를·이/가)는 지점명에 맞춰 자동 보정됩니다.

## 7. 일괄 처리 (CSV Import / Bulk)

1. `/reviews/import` — CSV 업로드 (플랫폼 컬럼 매핑 지원, 지점 자동 감지).
   중복은 5차원 해시(지점|채널|작성자|날짜|본문)로 자동 차단 — 같은 파일 재업로드 안전.
2. 인입 즉시 전 건 자동 분류 — 긴급/불만은 그 순간 격리되므로 목록에서 위험도 순 확인.
3. `/reviews` 목록에서 Gmail식 전체 선택 → 일괄 AI 생성/일괄 보관.

## 8. 문제 발생 시

| 증상 | 조치 |
|---|---|
| 초안이 한국어로 나옴 (영어 리뷰인데) | 리뷰 상세에서 `review_language` 값 확인 → 잘못되어 있으면 수정 후 재생성 |
| 답변에 `{branch_name}` 같은 토큰이 그대로 보임 | 즉시 개발팀 보고 (P0 — 회귀 게이트 위반) |
| 멀쩡한 ★5 리뷰가 EMERGENCY 격리됨 | 리뷰 내 "환불/다쳤" 등 키워드 인용 여부 확인 → 오탐이면 검토 후 수동 승인, 패턴 개선 요청 |
| LLM 초안 생성 실패 | 자동으로 건조 사과 정적 초안 + 승인대기로 떨어짐 — 직접 작성하면 됨 |
| 게시 후 상태 변경 누락 | 상세 페이지에서 "게시 완료" 수동 처리 (활동 로그 기록됨) |

## 9. 개발/QA — 변경 시 검증 절차

```bash
npx tsc --noEmit                                                  # 0 에러
npm run lint && npm run build                                     # 0 / 성공
npx tsx scripts/validate-waterfall.ts | tail -5                   # 116+ ALL PASS
npx tsx scripts/deep-learning-loop.ts 2>&1 | grep "이슈 있는"      # 0/813 필수
```

- DB 변경은 Supabase MCP 적용 + `supabase/migrations/NNN_*.sql` 커밋 동시 수행,
  `CLAUDE_CONTEXT.md` §4 표 갱신.
- **RLS STEP B**(`supabase/gated/`)는 명시적 "RLS 락 해제" 승인 전 적용 금지.

## 10. 감사 추적

등록/생성/승인/게시/보관/삭제 — 모든 주요 액션은 `activity_logs`에
타임스탬프 + 행위자 + 상세 JSON으로 영구 기록됩니다. 삭제는 soft delete(복구 가능)가 기본이며,
완전 삭제는 아카이브에서 관리자만 가능합니다.
