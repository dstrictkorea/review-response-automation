# Dashboard Acceptance Criteria

## Required

- high contrast text
- Korean-first UI
- visible risk badges
- filters work
- no publish button for manual channels
- no publish button before approval
- high-risk publish blocked
- copy button logs event
- edits are saved
- audit drawer visible
- SLA overdue badge visible

## Accessibility

- body text contrast high
- status not color-only
- buttons have labels
- keyboard navigation for approval actions
