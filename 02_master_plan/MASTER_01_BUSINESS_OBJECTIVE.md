# Business Objective

## 1. Why This Exists

아르떼뮤지엄 리뷰 대응 업무는 단순 CS가 아니라 브랜드 평판, 지점 운영 품질, 재방문 의도, 본부 보고의 데이터 원천입니다.

## 2. Business Goals

| Goal | Description | Metric |
|---|---|---|
| Response workload reduction | 반복적인 리뷰 답변 작성 시간 절감 | 담당자 주간 처리 시간 |
| Reply quality consistency | 지점/언어별 답변 품질 균일화 | QA score |
| Risk control | 부정/민감 리뷰 오답 방지 | high-risk auto-publish 0 |
| Faster response | 신규 리뷰 대응 지연 감소 | median response time |
| Monthly insights | 지점별 고객 불만과 긍정 포인트 구조화 | monthly report completion |
| Brand localization | 다국어 고객에게 자연스러운 응대 | language QA pass rate |

## 3. Non-Goals

- 리뷰를 조작하지 않는다.
- 고객에게 리뷰를 쓰게 유도하거나 보상하지 않는다.
- 모든 채널을 무리하게 자동화하지 않는다.
- 플랫폼 정책을 우회하지 않는다.
- AI가 독단적으로 보상/환불/법적 입장을 제시하지 않는다.

## 4. Target Outcome

1. Google 리뷰는 수집부터 승인 후 게시까지 이어진다.
2. TripAdvisor/Naver는 수집과 초안 생성까지만 자동/반자동화한다.
3. 국내 OTA는 “수동 붙여넣기 → AI 초안 → 복사 게시”로 처리 시간을 줄인다.
4. 모든 리뷰 데이터는 월간 지점/본부 리포트로 자동 집계된다.
