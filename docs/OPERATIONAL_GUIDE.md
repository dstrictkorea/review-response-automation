# ARTE Museum — 리뷰 대응 운영 가이드

## 개요

이 시스템은 ARTE Museum 직원이 고객 리뷰에 대응하는 내부 운영 장부입니다.  
**자동으로 공개 답변을 게시하지 않습니다.** 모든 답변은 직원이 직접 검토하고 승인한 뒤 외부 플랫폼에 게시합니다.

---

## 역할 구분

| 역할 | 설명 | 권한 |
|------|------|------|
| `marketing_staff` | 마케팅 담당 직원 | 즉시 반영 + 관장 결재 요청 |
| `director` | 지점장 / 관리자 | 전결 승인 게시 + 본사 이관 + 고급 처리 |

---

## marketing_staff 운영 흐름

### 1단계 — 리뷰 등록
- 메뉴: **리뷰 등록** (`/reviews/new`)
- 고객이 남긴 리뷰 텍스트를 붙여넣고, 지점·채널·별점을 선택 후 등록

### 2단계 — AI 초안 생성
- 리뷰 상세 페이지에서 **AI 초안 생성** 버튼 클릭
- AI가 3가지 톤(표준/짧게/조심스럽게)의 답변 초안을 생성합니다
- 위험 표현(환불 약속, 법적 책임, CCTV 언급, 직원 징계) 자동 감지

### 3단계 — 답변 편집
- 원하는 톤 탭 선택 → **편집란에 적용** 버튼으로 복사
- **최종 답변 편집** 란에서 직접 수정

### 4단계 — 주요 액션 선택

#### 🟢 즉시 반영 (리스크 없는 일반 리뷰)
- **Google 채널**: `🟢 검토 완료 — Google 즉시 반영` 버튼 → Google Business Profile에 직접 게시
- **기타 채널**: `📋 답변 복사 및 외부 플랫폼 이동` 버튼 → 클립보드 복사 + 플랫폼 관리자 페이지 오픈

#### 🚨 관장 결재 요청 (리스크·환불 케이스)
- 법적 위협, 환불 요구, 고위험 민원 등 — `🚨 관장 결재 요청 (리스크·환불)` 버튼
- 상태가 `escalated`로 변경되며 지점장에게 처리 이관

---

## director 운영 흐름

### 주요 액션

#### ✅ 지점장 전결 승인 및 게시
- 직원이 작성한 답변을 검토 후 즉시 승인·게시 처리
- 승인(approved) + 게시 완료(manual_published) 상태로 동시 변경

#### 🏢 본사(HQ) 최종 이관
- 본사 수준 판단이 필요한 케이스 — 에스컬레이션 처리
- 상태: `escalated`

### 고급 처리 옵션 (director 전용)
| 버튼 | 설명 |
|------|------|
| 승인 | 답변 내용 승인 (published는 별도) |
| Google에 직접 게시 | Google Business Profile API로 게시 |
| 답변 복사 + 관리자 이동 | 비Google 채널 클립보드 복사 |
| 게시 완료 처리 | 수동 게시 후 상태 완료 처리 |
| 답변 불필요 | no_reply 처리 |
| 에스컬레이션 | HQ 이관 |
| 상태 되돌리기 | 이전 단계로 롤백 |

---

## AI 격리(pending_approval) 처리

AI가 다음 조건을 감지하면 자동으로 `pending_approval` 상태로 격리합니다:

- 위험도 `medium` 이상
- 금지 표현 감지: 환불 약속, 법적 책임 인정, CCTV 언급, 직원 징계 약속

**처리 방법**: 지점장(director)이 직접 검토 후 전결 승인 또는 HQ 이관

---

## 안전 수칙 (절대 준수)

1. **환불·보상 약속 금지** — 생성된 답변에 포함 시 즉시 수정
2. **법적 책임 인정 금지** — "저희 잘못입니다" 류의 표현 불가
3. **CCTV 확인 약속 금지** — 확인하지 않을 수도 있는 약속 불가
4. **직원 징계 약속 금지** — 내부 징계는 외부 답변에 언급 불가
5. **고위험 리뷰 반드시 검토** — AI 격리 배너가 뜨면 관장이 직접 검토

## SLA 기준

- 리뷰 작성 후 **3일 이내** 답변 처리 권장
- 3일 초과 시 리뷰 상세 페이지 상단에 ⚠ 경고 배너 표시

---

## 처리 이력

모든 주요 액션(등록, AI 생성, 승인, 게시, 에스컬레이션)은 타임스탬프와 함께 자동 기록됩니다.  
리뷰 상세 페이지 하단 **처리 이력** 섹션에서 확인할 수 있습니다.
