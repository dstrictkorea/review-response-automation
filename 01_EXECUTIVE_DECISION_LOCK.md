# 01. Executive Decision Lock

## 1. 최종 의사결정 문장

아르떼뮤지엄 리뷰 자동화 시스템은 **리뷰 수집·분류·AI 답변 초안·관리자 승인·채널별 게시 지원·월간 분석 리포트**를 통합하는 운영 시스템으로 구축한다.  
다만 **고객 리뷰 답변을 무검토로 자동 게시하는 시스템은 아니다.**

## 2. 확정해야 하는 큰 방향

| 항목 | 결정 |
|---|---|
| 운영 방식 | Human-in-the-loop |
| 1차 자동화 대상 | Google Reviews |
| 반자동 대상 | TripAdvisor, Naver |
| 수동+AI 초안 대상 | 국내 OTA/입점몰/기타 채널 |
| 우선 지점 | AMDB, AMNY, AMLV |
| 언어 우선순위 | 리뷰 본문 언어 > 지점 기본 언어 |
| 금지 방향 | 가짜 리뷰, 평점 조작, 무승인 자동 게시, 플랫폼 제한 우회 |
| 시스템 목적 | 답변 품질 표준화, 업무시간 절감, 리스크 통제, 리포트 자동화 |

## 3. 의사결정 근거

- 내부 보고서는 현재 해외 3개 지점과 국내 5개 지점의 다수 채널 리뷰 업무가 수동이며, 일별 리뷰 답변 자동화와 월간 리뷰 보고서 자동 생성을 목적으로 설정했다.
- 내부 보고서는 Google Reviews를 자동 수집·AI 초안·승인 후 게시 가능 대상으로, TripAdvisor/Naver를 반자동 수집과 수동 게시 대상으로, 기타 국내 채널을 AI 초안 중심 대상으로 분류했다.
- 공식 Google Business Profile API는 리뷰 조회와 답변 생성/수정 기능을 제공한다.
- TripAdvisor는 Content API가 있으나 공개적으로 사업자 답변 자동 게시 API로 보기 어렵고, 공식 안내는 Management Center에서 응답하는 흐름이다.
- Naver SmartPlace는 사업주가 답글을 작성할 수 있지만, 공개 답글 게시 API 기반 자동화는 검증되지 않았다.
- n8n은 self-hosting/Docker 설치가 가능하므로 회사 서버 기반 자동화 허브로 적합하다.

## 4. 이번 정책팩의 잠금 기준

아래 항목은 추후 구현 중에도 바꾸지 않는다.


1. **완전 무인 자동 답변봇으로 만들지 않는다.** 초기 운영은 전 채널 관리자 승인형이다.
2. **Google만 승인 후 자동 게시 후보**로 둔다. Google도 1~2개월 안정화 전에는 무조건 승인형이다.
3. **TripAdvisor/Naver/국내 OTA는 자동 게시 대상이 아니다.** 공식 게시 API가 검증되기 전까지 복사/붙여넣기 반자동이다.
4. **리뷰 작성/평점 조작/가짜 리뷰/보상형 리뷰 유도는 시스템 범위에서 영구 제외**한다.
5. **고객 국적을 추정하지 않는다.** 리뷰 본문 언어만 감지한다.
6. **한국어·영어·일본어·중국어·아랍어를 1차 전문 언어로 잠근다.**
7. **언어는 직역 금지.** 각 언어권의 자연스러운 비즈니스 응대 문장으로 작성한다.
8. **1~3점, 안전, 사고, 환불, 차별, 아동, 법무, 언론, 개인정보 리뷰는 자동 게시 금지**한다.
9. **답변은 항상 감사·공감·사실 범위·개선 의지 중심**이며 환불/보상/법적 인정/직원 징계 약속을 하지 않는다.
10. **모든 수집·분석·수정·승인·게시 이벤트는 감사 로그로 남긴다.**
11. **정책 변경은 문서 PR 또는 승인 로그 없이는 운영에 반영하지 않는다.**
12. **운영 지표는 답변 속도보다 답변 안전성과 브랜드 일관성을 우선한다.**


## 5. “정책 확정”의 의미

정책 확정은 모든 기능 구현 완료가 아니라, 아래 기준을 의미한다.

1. 무엇을 자동화할지/하지 않을지 정했다.
2. 어느 채널을 자동 게시 대상으로 둘지 정했다.
3. 어떤 리뷰가 반드시 사람 검토 대상인지 정했다.
4. 어떤 언어와 톤을 우선 지원할지 정했다.
5. 어떤 데이터 구조로 기록할지 정했다.
6. 어떤 운영 지표로 성공/실패를 볼지 정했다.
7. 어떤 테스트를 통과해야 다음 단계로 갈지 정했다.
